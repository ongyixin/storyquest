# StoryQuest

> *Type a premise. Five agents turn it into a living comic.*

StoryQuest is an agentic comic-story platform powered by a five-agent LLM pipeline. Paste a premise, pick a vibe, and watch a Director agent build the story bible, a Writer script the panels, a Continuity agent enforce cross-panel consistency, and an Art Director compose image prompts — all fed to DALL-E 3 for illustrated output. In Interactive mode, the story keeps going: branch with choices or speak directly to characters via voice, and new panels generate in real time.

- **Normal mode** — generate a complete 6-panel read-only comic.
- **Interactive mode** — make choices, talk to characters via voice, and shape the story.

## Quick Start

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment variables

```bash
cp .env.local.example .env.local
```

Edit `.env.local` and fill in your keys (see table below).
For a local demo without real API keys, leave everything empty and keep `DEMO_MODE=true`.

### 3. Set up Convex

```bash
npx convex dev
```

This will:
- Prompt you to create/link a Convex project.
- Set `CONVEX_DEPLOYMENT` and `NEXT_PUBLIC_CONVEX_URL` automatically in `.env.local`.
- Watch for schema/function changes and hot-reload.

### 4. Start the Next.js dev server (in a separate terminal)

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## How It Works

### Agent Pipeline

Every generation request runs through a sequential five-agent pipeline. Each agent calls MiniMax M2.5 via `minimaxChat`, requesting structured JSON output validated against a Zod schema. Failures are retried up to 3 times with exponential back-off (1.5 s base, 120 s per-request timeout).

```
User premise + vibe
        │
        ▼
┌─────────────────────────────────────────────────────────┐
│  Director Agent                                          │
│  Reads premise + vibe, outputs:                         │
│   • storyBible  (genre, setting, arc, rules, tone)      │
│   • styleGuide  (line style, palette, camera rules)     │
│   • characterSheets  (appearance anchors, constraints)  │
│   • panelOutline  (6 beats, locations, emotional tones) │
│   • [interactive] worldState + initial choiceSet        │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│  Writer Agent                                            │
│  Converts panelOutline into full PanelScript[]          │
│   • sceneDescription, camera, dialogue (≤2 bubbles,     │
│     ≤20 words each) per panel                           │
│   • Dialogue is text only — no visual embedding         │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│  Continuity Agent                                        │
│  Reviews scripts against storyBible + worldState        │
│   • Catches location jumps, missing props, tone breaks  │
│   • Returns corrected scripts (status: "ok" | "fixed")  │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│  Art Director Agent                                      │
│  Produces one imagePrompt per panel                     │
│   • Merges styleGuide + characterSheet appearance       │
│     anchors into a single DALL-E-ready prompt           │
│   • Explicitly prohibits text-in-image                  │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│  DALL-E 3 (OpenAI Images API)                           │
│  All panel images generated concurrently                │
│   • 1792×1024 landscape (16:9 webtoon format)           │
│   • Individual failures return "" without aborting      │
└─────────────────────────────────────────────────────────┘
```

### Interactive Mode Extensions

When mode is `interactive`, two additional flows branch from the base pipeline:

**Choice flow** (reader picks a story option):
```
Reader selects choice
  → Director (after-event) → updated worldState + nextSceneGoal + new choiceSet
  → Writer → Continuity → Art Director → DALL-E 3
  → 1–2 new panels appended to the session
```

**Voice flow** (reader speaks to an NPC):
```
Speechmatics STT → transcribed text
  → NPC Agent → replyText + recommendedStateDelta (character-consistent response)
  → Director (after-event) → updated worldState + nextSceneGoal
  → Writer → Continuity → Art Director → DALL-E 3
  → NPC voice reply (Vapi TTS) + 1–2 new panels
```

### Structured Output Strategy

All LLM calls use a prompt-embedded JSON Schema approach rather than the `response_format` API parameter (which is unreliable across model versions). The full JSON Schema is appended to the user prompt as a format contract. Responses are extracted with a fence-stripping JSON parser, then validated with Zod. Zod validation failures are treated as retryable errors so the model gets a second chance before the request fails.

### Realtime State (Convex)

Session state is stored in Convex and pushed to the client over a live subscription — no polling needed. The database schema includes:

| Table | Purpose |
|-------|---------|
| `sessions` | Mode, premise, vibe, status, storyBible, styleGuide, characterSheets, worldState |
| `panels` | Per-panel script, imagePrompt, imageUrl, panelIndex, pageNumber |
| `choiceSets` | Branching options anchored to a panel index |
| `events` | Immutable log of choices and voice utterances |
| `conversations` | NPC conversation history per session + character |

Convex actions (server-side async functions) orchestrate the full pipeline: they call `generate.ts`, receive assembled panels + metadata, write everything to the DB in a single mutation, and the frontend reacts immediately.

### MiniMax Client (`src/lib/llm/minimax.ts`)

The `minimaxChat<T>` generic function is the single integration point for all LLM calls:

- Accepts a `system` prompt, `user` prompt, `jsonSchema`, and a Zod `parse` function
- Appends the JSON Schema to the user message as an inline format constraint
- Applies a 120 s `AbortController` timeout per attempt
- Retries on 5xx / 429 / network errors with exponential back-off (max 3 attempts)
- Falls back to `reasoning_content` when `content` is empty (MiniMax reasoning models)
- Returns a fully validated, typed `T` or throws with a descriptive error

## Agent Integration Points

| Agent | File to modify | What to add |
|-------|----------------|-------------|
| Agent 1 (Convex) | `convex/sessions.ts` | Real DB mutations, file storage for images |
| Agent 2 (Pipeline) | `src/lib/server/generate.ts` | Replace `TODO` stubs with real LLM pipeline calls |
| Agent 3 (Voice) | `src/components/VoiceBar.tsx` | Wire Speechmatics STT + Vapi TTS |
| Agent 4 (UI/Art) | `src/components/ComicGrid.tsx` | Style, panel transitions, image rendering |

**Shared contracts are frozen** — do not change `types.ts`, `schemas.ts`, or `contracts.md` without a team sync.

## Demo Instructions

1. Make sure `DEMO_MODE=true` in `.env.local`.
2. Run `npx convex dev` (needs Convex account).
3. Run `npm run dev`.
4. Open [http://localhost:3000](http://localhost:3000).
5. Enter any premise (≥ 10 characters), pick a vibe and mode, hit **Generate Comic**.
6. The session page will appear with 6 placeholder panels (demo stubs, no real API calls).
7. In **Interactive** mode, choice buttons and the voice text box will be active.
